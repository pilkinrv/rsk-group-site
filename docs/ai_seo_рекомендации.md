# Оптимизация под поиск ИИ-моделями (AI SEO)

> Рекомендации по обеспечению индексации и корректного ответа ИИ-поисковиками (Perplexity, ChatGPT Search, Google AI Overview и др.).

## 1. Структурированные данные (JSON-LD)

### Обязательные схемы

**Organization** — на каждой странице:
```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "ООО РСК ГРУПП",
  "url": "https://рскгрупп.рф",
  "logo": "https://рскгрупп.рф/logo.png",
  "description": "Производитель светодиодного осветительного оборудования. Собственное производство в Казани, гарантия 5 лет.",
  "address": {
    "@type": "PostalAddress",
    "streetAddress": "Теплично-Комбинатская ул., д. 5В",
    "addressLocality": "Казань",
    "addressRegion": "Республика Татарстан",
    "postalCode": "420036"
  },
  "telephone": "+7 960 079-68-53",
  "email": "mail@rsc-groupp.ru",
  "openingHours": "Mo-Fr 08:00-17:00"
}
```

**Product** — на страницах каталога и карточках товаров:
```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Название светильника",
  "description": "Описание продукции",
  "brand": { "@type": "Brand", "name": "РСК ГРУПП" },
  "manufacturer": { "@type": "Organization", "name": "ООО РСК ГРУПП" },
  "category": "Наружное освещение"
}
```

**BreadcrumbList** — на внутренних страницах.

**FAQPage** — для страниц с FAQ (гарантии, доставка).

## 2. Семантическая разметка HTML

- Использовать `<article>`, `<section>`, `<nav>`, `<header>`, `<footer>`
- Иерархия заголовков: один H1, логичная цепочка H2–H6
- Атрибуты `aria-*` для доступности (помогают и ИИ)

## 3. Контент для ИИ

- **Факты и цифры:** «гарантия 5 лет», «изготовление от 21 дня», «собственное производство»
- **Чёткие формулировки:** избегать двусмысленности
- **Ответы на типичные вопросы:** «Что производит РСК ГРУПП?», «Где находится производство?»
- **Краткое резюме** в начале ключевых страниц (1–2 абзаца)

## 4. Meta и Open Graph

- Уникальные `title` и `description` для каждой страницы
- Open Graph для соцсетей и превью в чат-ботах
- Canonical URL для избежания дублей

## 5. Технические аспекты

- **Sitemap.xml** — полный список URL
- **robots.txt** — разрешения для краулеров
- **Быстрая загрузка** — ИИ-краулеры учитывают производительность
- **Мобильная адаптация** — обязательна

## 6. Человекочитаемый + машиночитаемый контент

Ключевая информация должна быть:
1. В видимом тексте на странице
2. В JSON-LD (дублирование для машин)
